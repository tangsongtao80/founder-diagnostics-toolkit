# Innovation Accounting（创新核算）

来自《精益创业》埃里克·莱斯。创新核算是一套为创业公司定制的指标框架——传统财务报表看的是过去，创新核算看的是学习速度。它回答了投资人/管理层最核心的问题：「我们怎么知道这个新业务在进步？」

## 核心理念

> "对创业公司来说，唯一重要的指标是学习速度——单位时间内验证了多少假设。"

传统企业用收入/利润考核新业务，但初创期这些指标没有意义。创新核算用三个学习里程碑替代。

## 适用场景

- 公司内部孵化新业务，不知道用什么 KPI 考核
- 创业早期需要向投资人证明进展
- 团队不知道自己到底在成长还是原地打转
- 想给不确定性高的项目建立客观评估标准

## 三个学习里程碑

### 里程碑 1: 建立基线
用 MVP 收集第一批真实数据，建立起点基线。
- 关键：不要美化数据，诚实记录现状
- 输出：转化漏斗各环节的基线数据

### 里程碑 2: 优化引擎
从基线出发，通过 Build-Measure-Learn 循环优化关键指标。
- 关键：每次实验改变一个变量
- 目标：将关键指标从基线推向理想状态

### 里程碑 3: 判断是否可规模化
当关键指标接近理想值，判断业务是否可以规模化投入：
- 如果靠近理想值 → 加大投入，规模化
- 如果多轮优化后进展缓慢 → 考虑转型

## 执行步骤

### 步骤 1: 定义创新核算的指标体系
选择一个三级指标框架：
- **一级指标**（北极星）：产品对用户的核心价值（如 DAU、GMV）
- **二级指标**（驱动）：驱动一级指标的关键行为（如注册转化率、留存率）
- **三级指标**（诊断）：出现问题时排查用的细化指标

### 步骤 2: 建立基线
跑一个小规模实验，收集每个指标的当前值。

### 步骤 3: 设定学习里程碑
为每个关键指标设定三个目标值：
- 基线值（当前）
- 及格值（证明方向正确的最低门槛）
- 理想值（可以规模化投入的触发点）

### 步骤 4: 定期汇报
建立月度或双周的学习报告，回答三个问题：
- 我们学到了什么？
- 关键指标的变化是多少？
- 下一个实验是什么？

## 输出格式

```
## 创新核算看板

### 北极星指标
- 指标：
- 当前基线：
- 及格值：
- 理想值：
- 当前状态：（离目标还有多远）

### 驱动指标
| 指标 | 基线 | 当前值 | 变化 | 目标 |
|:---|:---|:---|:---|:---|
| | | | | |

### 本周期学习
- 做的实验：
- 学到什么：
- 下一步：
```

## 注意事项

- 创新核算 ≠ 不考核——只是用学习指标替代财务指标
- 每一个里程碑都必须有明确的数字，不能靠感觉判断
- 如果连续 3 个学习周期关键指标没有改善，严肃考虑转型
