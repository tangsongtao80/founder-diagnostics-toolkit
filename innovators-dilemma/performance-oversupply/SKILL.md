# Performance Oversupply（性能过剩判断）

来自《创新者的窘境》克莱顿·克里斯坦森。当技术能提供的性能超出市场实际需求时，竞争基础会转移——从功能性→可靠性→便捷性→价格。如果你还在原先的维度上竞争，就是在打一场已经过时的战争。

## 核心理念

> "性能过度供给，是推动产品生命周期由一个阶段向另一个阶段过渡的一项重要因素。"

客户对性能的需求是有上限的。当你的产品已经「足够好」，继续提升性能不再产生价值——客户开始关心别的东西。

## 竞争基础演变四阶段

### 阶段 1：功能性竞争
- 核心问题：「这东西能做到什么？」
- 赢家特征：技术最强
- 何时转移：当产品对大多数客户「够用」了

### 阶段 2：可靠性竞争
- 核心问题：「这东西稳定吗？」
- 赢家特征：品质最稳定
- 何时转移：当可靠性变成「理所当然」

### 阶段 3：便捷性竞争
- 核心问题：「这东西好用吗？」
- 赢家特征：体验最好、最方便获得
- 何时转移：当便捷性也不再是差异化

### 阶段 4：价格竞争
- 核心问题：「为什么选贵的？」
- 赢家特征：成本效率最高
- 这是破坏性技术最喜欢切入的阶段

## 诊断清单

检查你所在市场的当前阶段：

- [ ] 产品功能对 80% 的客户已经「足够好」了吗？
- [ ] 高端客户还在乎的功能升级，主流客户已经不在乎了吗？
- [ ] 是不是只有评测和发烧友在讨论性能参数？
- [ ] 便宜的产品是不是在快速抢市场？
- [ ] 你的研发投入在增加但客户支付意愿在下降？

上述越多勾选，说明竞争基础转移越迫切。
