# 目标体系一致性诊断

来自《复盘》沈磊。核心发现：大多数「执行问题」其实是「目标体系不一致」——你的目的、目标、策略、指标彼此打架。

## 核心理念

> "目的和目标为一组，简称蓝图组；策略和策略指标为一组，简称实施组。"

蓝图组回答「为什么要做」和「做到什么程度」，实施组回答「怎么做」和「怎么追踪」。两组之间必须有一条清晰的因果链：如果实施组的策略执行到位，蓝图组的目标就一定能达成。

## 四层一致性检查

### 层 1：目的（为什么做）
- 这是最终想要的结果，不是手段
- 目的应该回答「如果这件事做成了，世界会有什么不同」
- ❌ 「我们要做一场营销活动」→ 这是手段不是目的
- ✅ 「我们要在目标客户中建立品类认知」→ 这是目的

### 层 2：目标（做到什么程度）
- 目标必须是目的的量化翻译
- 如果目的变了，目标必须跟着变
- ❌ 目的变了但目标没变 → 最常见的不一致

### 层 3：策略（怎么做）
- 策略是「选择」，不是「清单」
- 策略必须回答「为什么选这条路而不是那条」
- ❌ 「我们要做抖音、小红书、公众号」→ 这是清单不是策略
- ✅ 「我们选择小红书作为冷启动主阵地，因为目标客户在该平台决策链路最短」→ 这是策略

### 层 4：指标（怎么追踪）
- 指标是策略执行情况的信号灯
- 指标必须有预警线（低于多少要纠偏）
- ❌ 「关注粉丝增长」→ 太模糊
- ✅ 「小红书周新增精准粉丝 > 200，低于 100 则调整内容方向」

## 一致性检查矩阵

| 检查维度 | 问题 |
|:---|:---|
| 目的 ↔ 目标 | 目标达成 = 目的实现吗？ |
| 目标 ↔ 策略 | 策略执行到位 = 目标能达成吗？ |
| 策略 ↔ 指标 | 指标能反映策略执行情况吗？ |
| 指标 ↔ 目的 | 指标好 = 目的在达成吗？（警惕虚荣指标） |

## 诊断输出格式

```
## 目标体系一致性诊断

### 当前目标体系
- 目的：
- 目标：
- 策略：
- 指标：

### 一致性检查
| 链路 | 是否一致 | 问题 |
|:---|:---|:---|
| 目的→目标 | ✅/❌ | |
| 目标→策略 | ✅/❌ | |
| 策略→指标 | ✅/❌ | |
| 指标→目的 | ✅/❌ | |

### 修正建议
- 需要调整的层：
- 调整方案：
```
