# 双维度复盘

来自《复盘》沈磊，源自联想复盘模型。复盘必须同时看两条线：逻辑线（数字说了什么）和行动线（人做了什么）。

## 核心理念

> "逻辑维度关注数字，是宏观层面的数字管理，重在检核进度、发现问题；行动维度关注动作，是微观层面的执行力管理，重在探讨细节、检视逻辑。"

只复盘数据 → 知道「什么出问题了」，不知道「为什么出问题」。只复盘行动 → 知道「谁做了什么」，不知道「影响有多大」。双线合在一起才有完整的复盘。

## 逻辑维度：数字管理

### 检核进度
- 关键里程碑达成了吗？
- 进度偏差是多少？趋势是恶化还是好转？

### 发现问题
- 哪些数字异常？（远高于/远低于预期）
- 异常数字之间有没有关联？（比如转化率降 + 客诉升）
- 是单点问题还是系统性问题？

## 行动维度：执行力管理

### 探讨细节
- 关键动作真的被执行了吗？
- 执行的「质量」如何？（做了 ≠ 做好了）
- 有没有人被「流程」捆住了手脚？

### 检视逻辑
- 当初的策略假设还成立吗？
- 如果策略没问题但结果不对 → 执行问题
- 如果执行没问题但结果不对 → 策略问题

## 双线交叉诊断

| 逻辑线结果 | 行动线结果 | 判断 |
|:---|:---|:---|
| 数字达标 | 动作执行到位 | 🟢 策略正确 + 执行到位 |
| 数字达标 | 动作没做到位 | 🟡 可能是侥幸或外部红利 |
| 数字不达标 | 动作执行到位 | 🔴 策略有问题 |
| 数字不达标 | 动作没做到位 | 🟡 先解决执行，再看策略 |
